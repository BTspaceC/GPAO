# 工作流：/审计 (Simulate Grading)

## 适用场景

作业即将提交前，进行最终审查。

## 必须输入

1. 完成的作业文稿（PDF、Word或Markdown）
2. 评分标准
3. 已填写的评分点矩阵（`../templates/rubric_visibility_matrix.md`）
4. 使用者本人的写作样本（如有，用于风格对照）

## 必须输出

### 1. 30秒扫视模拟

只查看：标题、摘要/项目简介、目录、一级标题、图表、结论。

逐项回答：

| 检查项 | 通过/未通过 | 具体问题 |
| :--- | :---: | :--- |
| 老师能否在30秒内知道作业做了什么？ | | |
| 能否看见工作量？ | | |
| 能否看见最重要的结果？ | | |
| 能否看见课程知识被实际使用？ | | |
| 是否存在一个容易记住的亮点？ | | |
| 正文是否明确声明了附录中有哪些可复现的证据？ | | |

### 2. 3分钟浏览模拟

重点查看：研究问题/目标、方法、核心结果、图表解释、结论、局限性。

逐项回答：

| 检查项 | 通过/未通过 | 具体问题 |
| :--- | :---: | :--- |
| 各章节是否围绕同一个问题？ | | |
| 方法是否与问题匹配？ | | |
| 方法选择是否有解释（为什么用它）？ | | |
| 图表是否有紧跟的文字解释和结论？ | | |
| 结果是否能支持结论？ | | |
| 结论是否超出了数据范围？ | | |
| 是否有堆砌但没解释的部分？ | | |

### 3. 深度复核

| 检查项 | 通过/未通过 | 具体问题 |
| :--- | :---: | :--- |
| 表格数值与正文段落描述是否一致？ | | |
| 参考文献是否全部真实、可查证？ | | |
| 代码/结果能否从原始数据复现？ | | |
| 附件是否完整、命名是否清楚？ | | |
| 是否满足所有格式和提交要求？ | | |

### 4. 附录证据链完整性

| 检查项 | 通过/未通过 | 具体问题 |
| :--- | :---: | :--- |
| 正文是否说明了附录内容？ | | |
| 附录有清晰编号和索引？ | | |
| 附录材料能支持结论复现？ | | |
| 附录是否只含必要证据（无填充）？ | | |

### 5. 学生表达检查

| 检查项 | 通过/未通过 | 具体问题 |
| :--- | :---: | :--- |
| 是否存在空洞的抽象评价？（例如"取得了良好效果"） | | |
| 是否存在过度拔高结论？ | | |
| 是否存在连续的工整排比句？ | | |
| 是否存在超出使用者理解范围的概念？ | | |
| 是否使用高频模板词？（可选增强：如本地存在 `../tools/student_voice_auditor.py`，则可运行辅助；否则依赖人工判断，不影响主流程） | | |
| 是否有提取并呈现真实过程证据？（注意：只能写真实经历，不得编造） | | |

### 6. 冲突检查

是否存在评分标准、老师偏好和数据条件之间的冲突？如有，是否按 SKILL.md 的冲突优先级处理？

### 7. 总体风险评估

列出不超过5个最关键的风险，按影响从高到低排序。

## 辅助 Prompt：去空洞化与学生表达重写

当需要AI辅助修改空洞表达时，可使用以下模板：

```
请修改以下段落。要求：
1. 将抽象评价替换为具体的数据、操作或结果
2. 不使用"诚然""不可否认""值得注意的是""综上所述""双刃剑""扮演了至关重要的角色""在宏大背景下"
3. 保留原文的逻辑和事实，只修改表达方式
4. 语气应像认真完成作业的学生，不像百科全书
5. 不要故意写错或降低质量来伪装
6. 不要添加使用者没有提供的事实或经历

原文：
[粘贴]
```

## 辅助 Prompt：局限性撰写

当需要撰写"局限性与不足"章节时：

```
请根据以下研究信息，撰写一段局限性说明。要求：
1. 只描述使用者提供的真实限制条件，不得编造未发生的困难
2. 对每个限制，说明已经采取了什么应对措施
3. 不自我贬低，不过度道歉
4. 语气平实、学术

研究信息：
- 样本情况：[填写]
- 方法限制：[填写]
- 时间/资源限制：[填写]
- 使用者真实遇到的问题：[填写，没有就留空]
```

## 禁止行为

- 不声称本次审计能承诺特定分数或确保高分
- 不把工具的模板词检测结果解读为"AI生成概率"
- 不编造使用者没有经历过的"真实过程"来增加学生感
