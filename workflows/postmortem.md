# 工作流：/复盘 (Postmortem)

## 适用场景

成绩出来后，结合最终得分和老师反馈进行复盘。

## 必须输入

1. 最终得分
2. 老师评语或反馈（如有）
3. 评分标准
4. 已提交的作业（如果还保留）
5. 之前填写的评分点矩阵（如有）
6. 之前的老师偏好假设（来自 `../templates/teacher_evidence_ledger.md`）

## 必须输出

### 0. 作业类型与适配器加载
（输出需包含：当前作业类型、加载适配器、识别依据、识别置信度）

### 1. 总体判断

用简洁语言说明：最终得分与现有证据是否一致，以及该判断的置信度、最主要的问题是什么。

区分两种情况：
- **真实质量问题**：内容确实存在不足（方法错误、数据不可靠、结论有误）
- **评分可见性问题**：工作完成了但老师没有看到（附件未展示、摘要缺少关键信息）

不得将所有低分原因归咎于老师压分。

### 2. 评分项逐项分析

| 评分项目 | 分值 | 正文可见证据 | 可能得分 | 推断依据 | 证据等级 |
| :--- | :---: | :--- | :---: | :--- | :---: |
| | | | | | 高/中/低 |

### 3. 老师偏好验证

对照 `../templates/teacher_evidence_ledger.md` 中的之前假设：

| 之前假设 | 本次结果是否支持 | 更新后可信度 | 更新后验证状态 |
| :--- | :---: | :---: | :---: |
| | 支持/推翻/无法判断 | | |

### 4. 下次作业改进策略

区分：
- **必须修改**：本次确认的高影响问题
- **建议修改**：中可信度的改进方向
- **需要更多证据**：本次无法确认的假设

每条策略标注：

| 改进策略 | 评分影响 | 可见概率 | 时间成本 | 证据可信度 | 优先级 |
| :--- | :---: | :---: | :---: | :---: | :---: |
| | 低/中/高 | 低/中/高 | 低/中/高 | 低/中/高 | P0-P3 |

### 5. 不确定性声明

列出本次复盘中无法确认的部分。例如：
- 老师是否查看了附件？（无法验证）
- 扣分是否因为方法简单还是格式问题？（无直接证据）

## 禁止行为

- 不保证按修改策略执行后能拉回高分或达到特定分数
- 不把用户的情绪判断（"老师就是压分"）转化为已验证的偏好事实
- 不编造老师没有给出的评语
- 当老师没有给出逐项分数时，不伪装精确的逐项得分
