# 工作流：/规划 (Plan Assignment)

## 适用场景

刚拿到任务书，尚未正式开始写作或开发时。

## 必须输入

1. 作业题目或任务书（A级必须）
2. 评分标准（A级必须）
3. 课程名称和作业类型（A级必须）
4. 当前已有成果（如有）
5. 老师偏好证据（如有，来自 `../templates/teacher_evidence_ledger.md`）

缺少评分标准时：标注"评分标准未提供，以下分析基于通用学术规范推断，存在较大不确定性"。

## 必须输出

### 0. 闭环适配器加载
- 解析当前作业类型，并加载对应适配器（输出需包含：当前作业类型、加载适配器、识别依据、识别置信度）。
- 如果无法识别，使用通用模式并声明不确定性；如果对应适配器缺失或损坏，停止类型专用判断，不能假装已加载。


### 1. 任务要求清单
列出任务书中所有明确要求，逐条标注是否已满足。

### 2. 评分点矩阵
按 `../templates/rubric_visibility_matrix.md` 格式，建立每个评分项的证据与可见性追踪。

### 3. 老师偏好假设（如有证据）
引用 `../templates/teacher_evidence_ledger.md` 中的记录，标注可信度。没有证据时不得编造偏好。

### 4. 任务优先级
按以下确定性决策矩阵分级，消除主观性：

- **P0**：满足任一条件（硬性提交要求未满足；数据、方法或结论存在真实性问题；文件无法打开或系统无法运行；核心评分项完全缺失；存在学术规范或引用风险；截止时间前必须完成）。
- **P1**：通常需要同时满足（对主要评分项有直接影响；有 FACT 或 HIGH 级证据支持；当前完成度不足或可见性风险较高；在剩余时间内可完成）。
- **P2**：包括（可拉开差异但不是基础合格条件；证据为 MEDIUM；高辨识度成果；在 P0 和 P1 完成后仍有时间执行）。
- **P3**：包括（主要影响视觉美化；与评分标准关系弱；时间成本高而预期收益低；依据仅为 LOW 级推测）。

每项任务标注优先级判断依据：

| 任务 | 对应评分项 | 评分影响 | 可见概率 | 时间成本 | 证据可信度 | 优先级 |
| :--- | :--- | :---: | :---: | :---: | :---: | :---: |
| | | 低/中/高 | 低/中/高 | 低/中/高 | 低/中/高 | P0-P3 |

*注意：人工覆盖决策表时必须说明覆盖原因。低可信度偏好不得产生 P0。没有正式评分权重时，不得使用“高分值”作为判定依据。真实性问题始终优先于可见性优化。*

### 5. 高辨识度成果设计（亮点）
设计1个容易被老师记住的亮点，需满足：
- 与课程内容相关
- 能真实完成
- 老师容易理解
- 可以在摘要、图表或演示中展示

可选择的亮点类型：稳健性检验、基准对比、异常处理记录、失败案例分析、真实案例测试、可复现代码、完整流程图、误差分析等。

### 6. 最小可交付版本
描述：如果只有有限时间，P0+P1完成后的最低可接受版本是什么。

### 7. 最终提交文件清单
列出需要提交的所有文件、格式和命名要求。

## 辅助 Prompt：防御性大纲生成

当需要AI辅助生成作业大纲时，可使用以下模板：

```
你是一位课程辅导助手。请根据以下信息生成一份作业大纲。

输入信息：
- 作业题目：[填写]
- 数据/样本情况：[填写]
- 评分标准：[粘贴]
- 老师已知偏好（含可信度）：[填写]

要求：
1. 大纲结构必须与评分标准中的每个计分项形成对应关系
2. 如果样本量或数据条件存在短板，在大纲中预留一节用于说明限制和应对措施
3. 不要使用"深入探讨""系统揭示"等空洞用语，标题应具体
4. 为每个核心评分项规划对应的图表或表格位置
5. 预留附录结构：原始工具/数据编码/分析代码或截图
6. 不要编造研究假设、数据或文献
```

## 禁止行为

- 不得在没有评分标准的情况下假装知道评分权重
- 不得为了增加P2任务而建议使用数据条件不支持的方法
- 不得把用户的情绪判断（如"老师就是压分"）当作事实写入规划
