# 适配器：实验报告（Experiment Report）

## 识别信号

- 作业围绕实际实验目的、仪器、操作步骤、原始读数、计算和误差分析展开。
- 材料包含测量单位、仪器分辨率、重复测量、理论值、拟合或不确定度。
- 核心成果是从真实操作和测量记录到结果解释的可追溯链条。

模拟实验、数据分析或编程控制若同时构成主要交付物，应声明混合类型并按需结合其他适配器，不强行归为单一实验报告。

## 反信号

- 没有实际或模拟实验过程，核心只是文献综述或理论推导。
- 核心是问卷、观察样本或社会科学数据的统计推断，应优先考虑实证类论文适配器。
- 核心是可运行系统、代码功能和工程测试，应优先考虑编程项目适配器。

## 必要提问

1. 任务书、实验指导书和 rubric 要求哪些章节、公式、图表与附件？
2. 实际使用了哪些仪器、量程、分辨率、步骤和安全条件？
3. 操作是否偏离指导书，哪些步骤是实际完成而非计划？
4. 原始读数、单位、重复次数、剔除记录和计算过程是否可核验？
5. 理论模型及误差/不确定度处理适用条件是什么？
6. 是否允许提交原始记录、照片、代码或表格，是否含敏感信息？

缺少原始数据时可审计结构与待补材料，但不得代填读数、异常或计算结果。

## 领域红线

- 不得编造、篡改或“校正”原始读数、异常现象、仪器参数、实验步骤、理论值、计算结果、图片或签字记录。
- **默认权重：禁止提供。** 只有任务书、rubric 或教师直接要求给出正式分值时才引用；不得编造评分权重或百分比。
- 不得为了贴近理论值删除数据或隐藏失败；任何剔除、重测和修正都必须有原始记录及规则依据。
- 误差方向、大小、传播方式和主导来源必须由仪器、模型与数据支持；信息不足时使用条件式解释，不确定结论不得写成事实。
- 不得把误差传播、线性拟合、显著性检验或某种绘图形式当作所有实验的无条件要求。
- 不照抄指导书冒充实际操作；若实际流程未知，标为待核实。
- 原始记录、照片和处理文件只有在提交规则允许、隐私合规且实际存在时才建议提供。

## 证据可见成果

- 实验目的、原理与所用公式，并说明适用条件和来源。
- 与真实操作一致的仪器信息、步骤和偏离记录。
- 带单位、有效数字和来源标记的原始数据表；重测或剔除保留理由与记录。
- 从原始读数到中间量和最终结果的可复核计算链。
- 图表的坐标、单位、图例和拟合方式与数据及任务要求一致。
- 在测量模型支持时报告不确定度或误差传播；否则给出具体、可证据化的定性限制。
- 异常和偏差分析说明发生环节、可能机制、方向与影响边界，不编造“真实感”故事。
- 在允许且合规时，可附原始记录、数据处理代码或表格，并在正文标明对应关系。

## 常见失败模式

- 将理论值当作应当复制的答案，反向修改测量数据。
- 无条件要求复杂误差传播，或为每个误差源生成虚假“贡献权重”。
- 只写“存在系统误差/人为误差”，没有连接到具体仪器、步骤和结果影响。
- 把计划步骤、指导书步骤和实际操作混为一谈。
- 图表缺少单位、有效数字不一致，或计算无法回溯到原始读数。
- 没有正式 rubric 时生成权重，或把不存在的附件写成已提交。

## 审计清单

- [ ] 类型判断有实验过程和测量证据支持，混合类型已声明。
- [ ] 原理、公式及其适用条件准确且有来源。
- [ ] 仪器、步骤、偏离、读数、重测和剔除均对应真实记录。
- [ ] 单位、有效数字、坐标轴和计算链前后一致。
- [ ] 误差方向、传播和改进建议没有超过现有证据。
- [ ] 异常与失败未被隐藏，也未为增强真实感而编造。
- [ ] 默认权重未生成；任何分值均能回指正式来源。
- [ ] 附件实际存在并符合提交、隐私和安全约束。
