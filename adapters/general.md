# 适配器：通用与混合类型（General）

## 识别信号

- 作业类型未知，材料不足以可靠选择某个领域适配器。
- 一个任务同时包含论文、实验、编程或其他主要交付物，单一适配器会遗漏关键要求。
- 任务属于现有三个适配器之外的类型，但仍需要按证据进行规划、审计或修改。

使用本适配器时必须明确说明“类型不确定”或“混合类型”，列出判断依据与待确认信息，不强行选择领域适配器。

## 反信号

- 任务书、rubric 和交付物已明确匹配实证类论文、编程项目或实验报告中的一种。
- 用户只是要求普通写作、翻译或闲聊，没有课程作业、评分对齐或提交审计需求。

## 必要提问

1. 最终交付物是什么，任务书和 rubric 的原文要求是什么？
2. 哪些部分由学生实际完成，哪些仍是计划或缺失材料？
3. 当前阶段是诊断、规划、审计、修改、画像还是复盘？
4. 有哪些来源可以支持事实、评分项、约束与完成状态？
5. 截止时间、格式、工具、能力、隐私和提交限制是什么？

## 领域红线

- 不得因类型不明而强行套用统计方法、实验误差、软件架构等类型特定建议。
- 不得编造事实、来源、已完成成果、引用、教师偏好或评分项。
- **默认权重：禁止提供。** 只有任务书、rubric 或教师直接要求给出正式分值时才引用；不得编造评分权重或百分比。
- 不确定处必须保留不确定性，说明需要何种材料才能升级判断。
- 缺失材料只降低相关部分的结论强度，不妨碍处理已知且有证据的部分。

## 证据可见成果

- 任务要求—现有证据—当前完成状态—缺口的通用追踪表。
- 已验证事实、推断、建议和待确认问题的清晰分离。
- 按正式 rubric 排序的成果清单；没有 rubric 时不生成权重，只按硬性约束、依赖和风险排序。
- 混合任务按交付物拆分子部分，并注明每部分适用的适配器或通用规则。
- 每项建议说明证据、条件和可验证的完成标准。

## 常见失败模式

- 为了给出具体答案而猜测作业类型，并输出不适用的领域方法。
- 材料不足时全面拒绝，遗漏可以安全完成的结构整理或证据盘点。
- 把常见课程惯例写成当前课程的正式要求。
- 用通用“亮点”堆砌内容，而没有连接任务要求、证据或可验证成果。
- 没有正式 rubric 时生成权重或预测精确分数。

## 审计清单

- [ ] 已明确标注未知或混合类型及其判断依据。
- [ ] 没有强行输出任何类型特定方法建议。
- [ ] 事实、推断、建议和缺口彼此可区分。
- [ ] 每项正式要求、权重和完成声明都有来源。
- [ ] 缺失材料只影响相关判断，已知部分仍被积极处理。
- [ ] 混合任务已按交付物拆分并选择适用规则。
- [ ] 默认权重未生成；任何分值均能回指正式来源。
