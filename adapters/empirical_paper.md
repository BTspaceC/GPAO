# 适配器：实证类论文（Empirical Paper）

## 识别信号

- 作业要求用观察、调查或二手数据回答研究问题，并呈现统计分析。
- 材料包含样本、变量、量表、假设、回归、相关、组间比较或模型检验。
- 核心成果是从数据到结论的可追溯证据链，而不是只综述文献。

若只有部分信号，明确类型不确定性；若同时具有实验报告或编程项目特征，使用 `general` 或声明混合类型，不强行归类。

## 反信号

- 任务只要求文献综述、概念论证或反思写作，没有数据分析。
- 核心是仪器操作、原始测量和误差传播，应优先考虑实验报告适配器。
- 核心是可运行软件、代码测试和系统文档，应优先考虑编程项目适配器。

## 必要提问

1. 任务书和 rubric 对研究问题、方法、结果及附件有什么明确要求？
2. 研究问题、假设、变量类型和样本关系是什么？
3. 数据来自哪里，是否已获得、脱敏并允许用于该作业？
4. 样本量、缺失值、异常值、分布和方差条件目前已知到什么程度？
5. 已实际完成哪些分析，能提供哪些代码、日志或输出作为证据？
6. 提交格式、附件数量、隐私与伦理限制是什么？

材料不足时只降级受影响的判断；仍可整理已知要求、证据缺口和下一步。

## 领域红线

- 不得编造、补齐或修改样本、问卷数、清洗记录、统计量、显著性、图表、引用或分析结果。
- **默认权重：禁止提供。** 只有任务书、rubric 或教师直接要求给出正式分值时才引用；不得编造评分权重或百分比。
- 统计方法必须由研究问题、变量尺度、样本关系、样本量、期望频数、分布、方差、缺失值和异常值等条件支持。条件未知时先提问或给条件式选项，不直接指定方法。
- 单个 Likert 题通常按有序变量考虑；多题合成量表只有在结构与信效度证据支持时，才考虑近似按连续变量处理。不得把这条经验规则当成无条件结论。
- 相关、普通回归或组间差异不能单独支持因果、风险阈值或干预建议。结论措辞不得超过研究设计与数据的识别能力。
- 不得为了显著性更换分析、删样本或隐藏不利结果；任何数据处理都必须有可核验依据。
- 原始数据、问卷和分析附件只有在提交规则允许、隐私合规且确有用途时才建议提供。

## 证据可见成果

- 研究问题—文献依据—假设—变量—分析—结论的对应关系。
- 数据来源、纳入与排除标准、实际样本流程及变量定义；只记录真实发生的清洗步骤。
- 与问题和数据条件匹配的描述性统计及核心分析结果。
- 在适用时报告效应量和置信区间；不以单个 P 值替代实质解释。
- 稳健性或敏感性检验只在存在合理替代设定、样本支持且实际执行时呈现。
- 对不显著或反预期结果给出证据支持的解释，并清楚标注局限。
- 在允许且合规时，可附脱敏原始数据、分析代码/日志或问卷来源；正文标明对应位置。

## 常见失败模式

- 先选熟悉或复杂的方法，再反向拼研究问题与数据条件。
- 把高阶模型、图形化或稳健性检验当作所有实证论文的必做加分项。
- 用“样本量较小”自动指定某一种简单检验，而不检查变量、设计和分布。
- 把学院模板或教师偏好误写成统计事实，或在没有 rubric 时生成权重。
- 将相关写成因果、将组间差异写成阈值，或用显著性掩盖很小的效应。
- 把计划中的分析、附件或清洗步骤写成已经完成的事实。

## 审计清单

- [ ] 类型判断有识别信号支持，反信号与混合类型已处理。
- [ ] 所有数据、样本、处理步骤、统计量和图表均可追溯到来源。
- [ ] 方法选择明确检查了研究问题、变量、样本关系及关键统计前提。
- [ ] Likert 处理有题项结构和信效度依据，未套用绝对规则。
- [ ] 因果、阈值与干预措辞未超过研究设计的支持范围。
- [ ] 效应量、置信区间和稳健性分析仅在适用且实际完成时呈现。
- [ ] 不显著、缺失和异常结果未被篡改或隐瞒。
- [ ] 默认权重未生成；任何分值均能回指正式来源。
- [ ] 附件符合提交、隐私与伦理约束。
