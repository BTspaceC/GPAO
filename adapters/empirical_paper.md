# 适配器：实证类论文 (Empirical Paper)

## 适用范围
社会科学、经济学、心理学等需要收集数据、进行统计分析以验证假设的课程论文。

## 核心评分点转换表

| 常见评分点 | 在 GPAO 中的等效要求 | 正文应体现的位置 |
| :--- | :--- | :--- |
| **研究问题与假设 (15%)** | 假设必须基于文献推导，不能凭空捏造。要有明确的研究对象。 | 引言最后一段或独立的"研究假设"节 |
| **数据收集与样本 (20%)** | 必须说明数据来源。如果是问卷，必须报告有效问卷数、筛选标准和基本人口学特征。 | 方法部分："数据来源与样本"小节 |
| **方法与模型 (25%)** | 选择模型时必须说明为什么不用更简单的模型。要有变量定义表。 | 方法部分："变量说明"与"模型构建"小节 |
| **实证结果分析 (25%)** | 必须有描述性统计、主回归/核心分析结果。最好有稳健性检验（亮点）。 | 结果部分，按逻辑分小节 |
| **结论与局限性 (15%)** | 必须根据数据结果得出结论。必须诚实报告真实存在的局限性，并说明补救措施。 | 讨论或结论部分 |

## 高辨识度亮点设计池 (P2 优先级)

1. **数据清洗纪实**：在正文中用半页篇幅记录剔除无效问卷/异常值的过程（如："剔除了作答时间少于1分钟的样本32份"）。这证明数据是真实的。
2. **描述性统计的图形化**：不用枯燥的表格，用分布图或箱线图直观展示核心变量的特征。
3. **稳健性检验**：用不同的模型设定或样本子集再跑一次数据，展示结果的一致性。
4. **失败预期探讨**：如果某个假设没有得到显著支持，不要篡改数据，而是在讨论中真实分析为何不显著（如：样本量不足、测量工具局限），这是极具学生真实感的加分项。

## 常见冲突处理

- **数据条件不足 vs 高级方法偏好**：如果样本量只有50，老师偏好"复杂多元模型"，必须降级为简单的相关或非参数检验，并在局限性中说明样本量限制。真实性 > 老师偏好。
- **显著性追求 vs 真实结果**：结果不显著时，禁止伪造显著性。应通过文献解释不显著的合理性。

## 附件要求

为了确保“证据链完整性”，必须提交：
1. `raw_data`：清洗前的原始数据（如脱敏的Excel）
2. `analysis_code_or_log`：统计软件的输出日志（SPSS的Viewer导出、Stata的do文件或R/Python代码）
3. `questionnaire_origin`：问卷原件（如为调查研究）
*(在正文对应处必须标注："原始数据和分析代码详见附件")*
