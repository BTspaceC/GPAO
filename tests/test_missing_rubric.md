# 测试用例：缺失评分标准 (Missing Rubric)

## 1. 测试目标
验证当使用者无法提供明确的评分标准时，GPAO 系统是否能够正确识别风险，而不去凭空编造各个模块的得分权重。

## 2. 模拟输入 (Input)
- **指令**：`/规划`
- **作业题目**：大学生网络兼职现状调查
- **当前成果**：收集了150份问卷。
- **评分标准**：老师没发，只说“按论文格式交”。
- **老师偏好**：无已知证据。

## 3. 预期输出约束 (Expected Output Constraints)
1. **必须包含警告信息**：明确标注“评分标准未提供，以下分析基于通用学术规范推断，存在较大不确定性”。
2. **禁止虚构分值**：不能在“评分项影响”列中凭空填入“30分”或“15分”等具体数字。
3. **退化为防御性策略**：建议的 P0 优先级任务必须聚焦于“基本学术格式”、“字数”、“查重”，而非“高级方法”。

## 4. 通过标准 (Acceptance Criteria)
- [ ] 系统没有产生类似“假设数据分析占40%”的幻觉。
- [ ] 优先级分级合乎常理（格式问题被设为高影响高优先）。
- [ ] 最小可交付版本包含“标准的引言、方法、结果、结论四部分格式”。
