# 测试用例：冲突要求处理 (Conflicting Requirements)

## 1. 测试目标
验证当作业的硬性要求、公开评分标准与老师在课堂上的随口偏好发生冲突时，系统能否严格按照《SKILL.md》中的冲突处理优先级链做出判断。

## 2. 模拟输入 (Input)
- **指令**：`/审计`
- **作业任务书（硬性要求）**：文献综述不少于 10 篇近三年外文核心期刊文献。
- **公开评分标准**：文献综述深度与广度占 20%。
- **老师口头反馈（偏好证据）**：“大家写综述不用太多，简单提两句目前国内研究就行了，把重点放在后面的实证部分。”（中可信度）
- **当前文稿**：学生为了迎合老师“不用太多”的口头偏好，只写了 3 篇国内文献。

## 3. 预期输出约束 (Expected Output Constraints)
1. **识别冲突**：明确指出当前文稿虽然迎合了老师口头偏好，但严重违反了“10篇外文核心文献”的硬性要求。
2. **应用优先级**：依据优先级规则 `作业硬性要求 > 老师明确补充要求 > 已验证老师偏好`，判定必须优先满足任务书的硬性标准。
3. **制定调和策略**：既要满足硬性数量要求，又要兼顾老师“重点在实证”的偏好。
   - 建议：将 10 篇外文文献整理成一个规范的对比表格（极大地压缩篇幅），只用一段简短的文字总结其局限性，迅速引出国内研究和实证部分。

## 4. 通过标准 (Acceptance Criteria)
- [ ] 成功拦截了单纯因为迎合偏好而导致挂科的风险。
- [ ] 准确引用了优先级规则。
- [ ] 提出的修改建议既满足了数量底线，也没有生成冗长的文字让老师厌烦。
