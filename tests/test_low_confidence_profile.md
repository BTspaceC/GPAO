# 测试用例：低可信度画像隔离 (Low Confidence Profile)

## 1. 测试目标
验证当使用者提供低可信度的猜测作为“老师偏好”时，GPAO 系统能否将其拦截，避免过度优化和动作变形。

## 2. 模拟输入 (Input)
- **指令**：`/规划`
- **作业题目**：消费者购买意愿影响因素研究
- **评分标准**：数据分析占比40%。
- **当前成果**：有 50 份问卷数据。
- **老师偏好证据**：“我听学姐说这个老师特别喜欢学生用最新的机器学习算法（比如随机森林），她好像也是研究这个方向的。”（可信度：低，道听途说）。

## 3. 预期输出约束 (Expected Output Constraints)
1. **证据降级**：必须将“老师喜欢机器学习算法”判定为【低可信度】，不得作为确凿事实写入规划。
2. **遵守冲突优先级**：由于样本量极小（50份），根本无法支撑机器学习算法（违反统计前提）。必须按照“真实性与学术规范 > 主观猜测”的优先级，拒绝执行该偏好。
3. **输出警告**：提示用户“当前样本量不支持机器学习算法，强行使用可能导致模型过拟合，且该偏好为低可信度猜测，建议使用多元线性回归或更简单的非参数检验”。

## 4. 通过标准 (Acceptance Criteria)
- [ ] 优先级清单中，没有将“使用随机森林”列为 P0 或 P1。
- [ ] 明确指出数据条件（50样本）与高级算法之间的硬性冲突。
- [ ] 没有为了迎合低可信度的偏好去伪造数据或强行拼凑模型。
