# GPAO

GPAO（Grade Point Alignment Optimizer）是一个面向大学课程作业的 Codex Skill。它根据任务书、评分标准和现有材料，帮助用户规划作业、检查评分项覆盖、审计提交风险，并在不编造事实的前提下进行最小修改。

当前版本：`V3.1.0-RC2`

## 核心能力

| 指令 | 用途 |
|---|---|
| `/诊断` | 判断作业类型、主要风险和材料缺口 |
| `/规划` | 根据 rubric 和硬性要求安排任务优先级 |
| `/审计` | 检查合规、方法、事实、评分覆盖和可见性 |
| `/修改` | 生成有证据依据的最小修改，默认只预览 |
| `/画像` | 在授权范围内整理教师反馈和课程内偏好 |
| `/复盘` | 根据实际成绩和反馈更新判断与后续策略 |

同时支持对应英文别名和自然语言触发。

## 安装

将仓库安装到 Codex skills 目录：

```bash
git clone https://github.com/BTspaceC/GPAO.git ~/.codex/skills/gpao
```

也可以将现有仓库目录链接到 `$CODEX_HOME/skills/gpao`。安装后重新打开 Codex。

## 使用

可以显式调用：

```text
使用 $gpao 诊断这份课程作业，并按证据给出下一步。
```

也可以直接使用 `/诊断`、`/规划`、`/审计`、`/修改`、`/画像` 或 `/复盘`。

## 作业适配

内置以下适配器：

- 实证论文
- 编程项目
- 实验报告
- 类型未知或混合任务的 general 兜底

作业类型已经明确时，材料缺失只降低相关判断，不会将类型降级为 general。

## 运行方式

- Codex 模块化模式：按需读取工作流、模板和适配器，推荐使用。
- 单文件兼容模式：使用 `dist/GPAO.bundle.md` 全量加载。

## 安全边界

- 不编造数据、引用、教师反馈或评分权重。
- 没有明确授权时，`/修改` 只提供预览，不覆盖原文件。
- 教师画像默认只在当前会话和课程范围内使用。
- 不承诺具体分数，也不把有限评测描述为普遍可靠性证明。

## 验证状态

当前版本通过仓库检查、86 个单元测试和 Codex Skill 官方验证器。V3.1 冻结评测中未观察到严重真实性、隐私、学术规范或授权违规；随后发现的适配器路由问题已完成定向修复和验证。该版本仍为 RC，不是 Stable。
